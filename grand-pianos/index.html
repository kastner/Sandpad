The Grand Piano Challenge

One prompt. Thirteen models. Each asked to synthesize a concert grand piano entirely in HTML — no libraries, no editing.

Twelve models received the identical brief via t3.chat. A thirteenth was tested in T3 Code (Codex) with a nearly identical prompt. None of the outputs were touched afterward.

The results were ranked across three dimensions: synthesis depth (harmonic modeling, envelope fidelity, reverb), keyboard range (octaves covered, sustain, pedal), and visual design. Each piano below is live — click the keys to play.

The range of results is remarkable: the best entry cites real acoustic physics papers and implements research-grade models. Others manage a single octave of triangle waves. And then there are the surprises.

The Prompt
Create a html only file that’s a piano keyboard. Synthesize the sound the best you can to simulate a grand piano

Design

The most technically ambitious entry in the entire set — by a wide margin. Full 88-key range (A0–C8), 36-voice polyphony management, multi-string unison detuning that scales with register, 10 partials with inharmonicity, hammer-noise burst + bass thunk, stereo panning by register, convolution reverb with early reflections, compressor, pressure-sensitive velocity, and glissando drag. Live voice and audio-state readouts. Beautiful dark-amber concert-hall aesthetic with Cormorant Garamond headings. On paper, the clear winner.

88 keys (full range) 36-voice polyphony Multi-string unison Stereo by register Convolution reverb Velocity + glissando Live voice readout
🙃
The twist: The keyboard is upside-down. Black keys on the bottom, white keys on top. Despite the most sophisticated synthesis in the set and a genuinely beautiful UI, the visual orientation is inverted. This was the one that inspired the whole experiment — trying other models to see if someone else would get it right.
